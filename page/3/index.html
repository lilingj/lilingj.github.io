《图解密码技术》笔记

学英语

  • cryptography：密码学。加密书写符号，安全传输消息的方法。
  • eavesdropper：窃听者
  • encrypt/decrypt：加密/解密
  • plaintext：明文
  • ciphertext：密文
  • confidentiality：机密性
  • cryptanalysis：密码分析、破译
  • cryptanalyst：破译者
  • symmtric：对称的
  • asymmetric：非对称的
  • hybrid：混合的
  • authentication：认证、身份认证
  • spoofing：伪装、欺骗
  • hash：古法语中的“斧子”
  • collision：碰撞
  • authentication：认证

    概念

  • 密码算法：加密、解密算法合在一起称为~
  • 完整性：或一致性。数据是正牌不是伪造的
  • 选择明文攻击（Chosen Plaintext Attack，CPA）：密码破译者能选择任意明文并得到其加密的结果
  • 隐蔽式安全性（security by obscurity）：公开加密算法，保密加密密钥
  • 密码算法：
    • 分组密码（block cipher）：只能加密固定长度的明文。加密任意长度的明文就需要迭代，迭代方法被称为分组密码的模式（mode）
      • DES、3DES
      • AES
    • 流密码（stream cipher）：对数据流进行连续处理的一类密码算法。
      • 一次性密码本
  • 分组密码的模式
    • ECB模式：Electronic CodeBook mode（电子密码本模式）
    • CBC模式：Cipher Block Chaining mode（密码分组链接模式）
    • CFB模式：Cipher FeedBack mode（密文反馈模式）
    • OFB模式：Output FeedBack mode（输出反馈模式）
    • CTR模式：CounTeR mode（计数器模式）
  • 密码劣化：随着计算机技术进步，以前被认为安全的密码会被破译
  • 密钥：
    • 对称密码的密钥、公钥密码的密钥
    • 消息认证码的密钥、数字签名的密钥
    • 用于确保机密性的密钥、用于认证的密钥
    • 会话密钥、主密钥：一次性、重复使用
    • 用于加密内容的密钥（Contents Encrypting Key，内容加密密钥）、用于加密密钥的密钥（Key Encrypting Key，密钥加密密钥）：一般会话密钥是CEK，主密钥是KEK
  • 密码可以看做是压缩技术：
    • 机密性压缩：加密
    • 完整性压缩：哈希
    • 认证的压缩：消息认证码
    • 不可预测性的压缩：用随机数种子对伪随机数赋予不可预测性

历史上的密码

  • 凯撒密码：通过平移字母表加密。暴力破解。
  • 简单替换密码：手动设置字母对应关系。频率分析破译。
  • Enigma：通过每日密码本的每日密码设置机器，再用机器加密。
  • 维纳斯密码：一次性密码本，通过随机数列与明文的xor操作来加密，无条件安全，理论上无法破译

解密

  • 暴力破解
  • 频率分析
  • 差分分析：改变一部分明文并分析密文如何随之改变
  • 线性分析：将明文和密文一些对应的比特进行XOR并计算其结果为0的概率。若密文足够随机，则概率应该为1/2。分析大幅偏离1/2的部分
  • 中间人攻击（man-in-the-middle attack）
  • 选择密文攻击：发送任意数据，服务器都会将其当做密文来解密并返回解密结果

常用密码技术

对称密码

一次性密码本

​ 使用XOR加密，不知道密码现实时间中无法破译。即使暴力破解出原文，由于所有组合都会试到，那也没办法从很多有规则的明文中选出真正的明文。需要安全地运输与原文等长的密钥，既然如此，何不直接运输明文。不实用。

DES

Data Encryption Standard，分组密码（block cipher）

基本结构：Feistel网络

对64比特明文加密，每7比特设置1比特错误检查位，密钥56比特。将数据分组为64比特的组，分别加密。

3DES

将DES重复三次，用三个密钥进行 加密 -> 解密 -> 加密 的过程。三个密钥都相同则

3DES退化成DES。加密相同解密不同，则此3DES称为DES-EDE2，三密钥都不同称为DES-EDE3 。速度慢，目前只有银行使用。

AES

Advanced Encryption Standard，分组密码。为了取代前任标准DES的新标准。最终将Rijndael算法确定为AES

公钥密码

public-key cryptography

基于数学上困难的问题来保证机密性

RSA

  • 加密：密文 = 明文E mod N

  • 解密：明文 = 密文Dmod N

  • 公钥：E 和 N 的组合

  • 私钥：D 和 N 的组合

  • 生成密钥对：求解 E 、D、N 三个数

    • N = p * q：p、q为两个很大的质数，例如512b的质数
    • L = lcm(p - 1, q - 1)：用来生成密钥对
    • E：
      • 1 < E < L
      • E、L 互质。即 gcd(E, lcm(p - 1, q - 1)) = 1
    • D：
      • 1 < D < L
      • E * D mod L =1

混合密码系统

hybrid crypto system

对称密码加密明文，公钥密码加密对称密码密钥

单向散列函数

one-way hash function，或称消息摘要函数（message digest function）、哈希函数、杂凑函数

输入：消息（message）或称原像（pre-image)

输出：散列值（hash value）或称消息摘要（message digest）、指纹（fingerprint）

性质：

  • 抗碰撞性（collision resistance）：越强越好
  • 单向性（one-way）：不能被反推

MD4、MD5

产生长度128b的散列值。已经不安全

SHA-1

输入长度最多264 - 1b，产生160b的散列值。已经不安全

SHA-2

输入长度最多2128 - 1b，包括SHA-256、SHA-384、SHA-384、SHA-512等。尚未被攻破。

SHA-3

Keccak算法作为SHA-3标准。

Keccak输入没有长度限制，可以产生任意长度散列值，为了配合SHA-2的散列长度，共设置了SHA3-224、SHA3-256、SHA3-384、SHA3-512四种版本。另规定两个可以输出任意长度散列值的函数（extendable-output function，XOF），分别死SHAKE128和SHAKE256

RIPEMD-160

产生160b的散列值。比特币中使用的就是~

消息认证码

Message Authentication Code，MAC

输入：任意长度的消息 + 发送者和接受者的一个共享的密钥

输出：固定长度数据，称为MAC值

实现：

  • 使用SHA-2之类的单向散列函数可以实现消息认证码
  • 使用分组密码实现
  • 其他方法

攻击方法：

  • 重放攻击（replay attack）
  • 密钥推测攻击

随机数生成器

密码技术中不可预测性的源泉

随机数的性质

  • 随机性（randomness）：弱伪随机数
  • 不可预测性（unpredictability）：无法通过之前生成的序列推测出下一个数，强伪随机数
  • 不可重现性：运行软件的计算机本身只具有有限的内部状态，而内部状态相同的情况下，软件必然只能生成相同的数。因此软件所生成的数列在某个时刻一定会出现重复。对于软件所生成的数列，其周期一定是有限的。凡是具有周期的数列，都不具备不可重现性。要生成具有该性质的数列，需要从不可重现的物理现象中获取信息，例如采集CPU的热噪声。具备该性质的随机数称之为真随机数

真随机数

通过硬件生成。这样的硬件设备称为随机数生成器（Random Number Generator，RNG）

伪随机数

通过软件生成。这样的软件称为伪随机数生成器（Pseudo Random Number Generator，PRNG）

种子（seed）：用来对伪随机数生成器内部状态进行初始化

线性同余法

linear congruential method

算法：

  • 第一伪随机数：R0 = (A * 种子 + C) mod M，A、C、M都是常量，且A、C都小于M
  • Ri + 1 = (A * Ri + C) mod M，i = 0、1、2……

说明：

  • 线性同余法不具备不可预测性，是弱随机数，不可用于密码技术
  • C语言库函数rand，以及Java的java.util.Random类都是用线性同余法写的，不能用于密码技术
单向散列函数法

通过单向散列函数可以编写出生成强伪随机数的生成器。

算法：将种子作为计数器初始值，每次计数器加一，并对当前数生成单向散列值，作为伪随机数输出

密码法

算法：将种子拆分成两部分使用，分别是计数器初始值和密钥，每次计数器加一，并对当前数使用密钥加密，密文作为伪随机数输出

证书

公钥证书（Public-Key Certificate，PKC）记有姓名、组织、邮箱、地址等个人信息，以及属于此人的公钥，并由认证机构（Authentication Authority、Certifyint Authority，CA）施加数字签名。简称证书（certificate）

Public-Key Infrastructure，PKI

一个总称

  • 用户：使用PKI的实体
  • 认证机构：颁发证书的实体
  • 仓库：保存证书的数据库

密钥的管理

生成密钥

  • 用随机数生成密钥
  • 用口令生成密钥

更新密钥

key updating

发送者和接受者定期同时改变密钥。用当前密钥的散列值作为下一个密钥。可以防止破译过去的通信内容机制，称之为后向安全（backward security）

配送密钥

  • 通过事先共享

  • 通过密钥分配中心

    • 中心化数据库保存所有人的密钥

  • 通过公钥密码

  • 通过 DIffie-Hellman密钥交换：通信双方可以通过交换一些可以公开的信息就能够生成出共享的秘密数字。这一秘密数字就可以作为对称密码的密钥

保存密钥

作废密钥

向证书颁发机构申请作废密钥，证书机构维护一张已作废密钥表

PGP

PGP（Pretty Good Privacy）是一款商业密码软件，不能自由使用。后自由软件基金会决定开发一个PGP的替代品，GunPG，简称GPG。

https://www.gnupg.org/

SSL/TLS

Secure Socket Layer/Transport Layer Security

以前是http协议直接通信，https是让http协议通过SSL/TLS通信

工作流程：使用伪随机数生成器生成会话密钥作为对称密码的密钥，使用对称密码加密通信消息，使用消息认证码保证信息的完整性，使用公钥密码配送会话密钥，使用证书来保证公钥密码的公钥的合法性。

TLS 1.0 相当于 SSL 3.1，SSL 3.0 已经不安全了。

TLS

TLS分为两层协议，下层是TLS记录协议（TLS record protocal），上层是TLS握手协议（TLS handshake protocal）。TLS握手协议又分为四个并列协议：握手协议、密码规格变更协议、警告协议、应用数据协议。

TLS记录协议

位于TLS握手协议的下层，负责使用对称密码对信息进行加密通信；负责消息的压缩、加密以及数据的认证。使用了对称密码和消息认证码。具体的算法和共享密码则是通过握手协议在服务器和客户端协商决定的。

TLS握手协议

握手协议

负责在客户端和服务器之间协商决定密码算法和共享密钥。基于证书的认证操作也是在这个协议中完成的。

密码规格变更协议

负责向通信对象传达变更密码方式的信号。当协议中途发生错误时，通过警告协议传达给对方

警告协议

比如消息无法解密是通知服务器。如果没有错误，通过应用数据协议来进行通信

应用数据协议

量子密码和量子计算机

量子密码利用了两个事实：

  • 从原理上来说，无法准确测出光子的偏振方向
    • 窃听者听到的内容可能不正确
  • 测量行为本身会导致光子的状态发生改变
    • 接受者可以判断通信是否被监听

如果量子密码进入使用领域，可以使用量子密码来实现一次性密码本，从而产生完美的密码技术。

量子计算机：一个粒子可以同时具有多种状态，如果一个粒子能够计算0、1两种状态，那么128个粒子就可以同时计算2128种状态。这就是一台超级并行计算机，密码破译者的终极工具

椭圆曲线密码

源自于求椭圆弧长的椭圆积分的反函数

实数域上方程：y2 = ax3 + bx2 + cx + d

有限域（23）上方程：y2 ≡ ax3 + bx2 + cx + d （mod 23）

包括：

  • 基于椭圆曲线的公钥密码
  • 基于椭圆曲线的数字签名
  • 基于椭圆曲线的密钥交换
